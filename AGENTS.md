# AGENTS.md — Kimi Cheat Sheet

Guidelines for AI assistants working on this repository.

## Project Purpose

A practical reference guide for Kimi Code CLI users. Emphasizes critical thinking and effective delegation over memorization of commands.

## Key Principles

1. **Accuracy First** — All commands must be verified against the official Kimi CLI
2. **Think Critically** — Content should help users understand WHEN to use features, not just HOW
3. **Stay Current** — Kimi CLI updates frequently; verify against latest documentation
4. **No Hype** — Focus on practical utility, not marketing language

## Working Rules

- If a prompt would require a major architectural deviation, stop and explain before proceeding.
- Keep changes and milestones separated into distinct commits.
- After each change or milestone, commit and push before starting the next one.
- Do not bundle unrelated work from different prompts into the same commit.

## Structure

- **Level 1-5**: Progressive learning path
- **Command Reference**: Quick lookup table
- **Best Practices**: Guidelines for effective use

## When Making Changes

- Verify commands with `kimi --help` before documenting
- Check official docs: https://moonshotai.github.io/kimi-cli/
- Maintain consistent formatting with collapsible sections
- Update "Last updated" date in README

## Target Audience

Developers who:
- Want to leverage AI assistance effectively
- Value understanding over memorization
- Need practical patterns for real work
